# 温婉0.3 更新日志

## 版本 0.3.4 - 打工交互优化

### 新功能

1. **打工前新增手机内确认弹窗**
   - 在公司点击“工位办公”后，不再直接开始打工
   - 现在会先弹出手机内确认框，允许选择本次打工时长
   - 可选时长为 1 到 12 小时，并会实时显示本次预计收入

### Bug修复

1. **修复打工完成后使用系统提示框的问题**
   - 打工结算后不再弹出浏览器系统 `alert`
   - 改为显示手机内风格的提示卡片，视觉效果与现有界面保持一致
   - 收益提示会展示本次打工时长和最终收入

2. **修复打工行为没有触发后续回复的问题**
   - 打工完成后现在会像其他地点行动一样，继续触发一段新的剧情回复
   - 回复会结合本次打工时长、结束时间和当前场景生成后续内容
   - 即使生成失败，也会有系统文本兜底，避免交互中断

3. **修复时间推进后的剧情仍混用旧时间上下文的问题**
   - 跳过时间、睡觉、打工等会先推进时间再生成剧情的链路，现在会显式携带推进后的时间快照
   - 避免提示词文本写的是新时间，但模型内部仍按旧时间判断学校状态、地点和事件触发
   - 让时间流逝后的剧情描述与实际游戏时间保持一致

4. **修复黄毛事件判定混用旧时间的问题**
   - 黄毛首登场、第二黄毛登场、当天是否出现等判定统一改为使用当前回合真实生效的时间
   - `firstMetDate` 等记录也会同步使用推进后的日期
   - 减少跨时段、跨天互动后黄毛事件晚触发或日期不对的情况

### 系统优化

1. **打工结算与时间推进逻辑统一**
   - 收益会按“每小时 100 元”与所选工时自动结算
   - 时间推进改为按照实际选择的打工小时数同步推进
   - 钱包记录时间也会和打工结束时刻保持一致

2. **行为规则提示词改为只发送当前阶段**
   - 好感度与堕落度相关行为规则改为运行时动态拼装
   - 只发送当前数值对应阶段的允许行为、接受度和隐瞒规则，不再整包发送所有阶段说明
   - 同时移除了提示词链路中已废弃的弧光描述，减少无效上下文

3. **当前阶段的允许行为改为显式列出**
   - 不再使用“前一阶段全部行为”这类需要模型自行补推理的写法
   - 高阶段会直接写出当前完整允许列表
   - 降低模型误解阶段继承关系的概率

## 版本 0.3.3 - 重新生成修复与酒馆提示词去重

### Bug修复

1. **修复重新生成时的状态回滚不完整问题**
   - 重新生成角色回复时，会恢复对应操作发生前的完整状态快照
   - 快照现在包含消息、身体状态、地点、推文、日历事件、摘要进度和游戏时间
   - 编辑消息后重新生成，也会基于正确的上下文重新推进，而不是混用后续状态

2. **修复酒馆角色卡把前端宿主代码送进提示词的问题**
   - 酒馆角色卡中的前端启动正则改为只处理 AI 输出，不再参与 prompt 注入
   - 避免 `sillytavern-host` 的 HTML / JS 代码在发送提示词时被一起带给模型
   - 同步更新了角色卡生成脚本和现成角色卡配置

3. **修复酒馆 Generate 路径的重复上下文注入**
   - 开启“优先使用酒馆 Generate”后，不再由前端重复拼接酒馆自己的角色卡、预设和世界书
   - 这部分改为交给 SillyTavern 当前生成链路自行注入
   - 前端自己维护的状态、摘要、记忆和额外提示词仍然会正常保留

4. **修复旧存档摘要进度恢复异常**
   - 读档时会重新归一化 `summaryCheckpoint`
   - 当旧存档的已保存进度比推导值更合理时，会优先保留有效进度
   - 减少读档后摘要重复生成或进度倒退的问题

### 系统优化

1. **重新生成链路的快照机制增强**
   - 普通用户发言和系统操作现在都会保存动作前快照
   - 对于重新生成、编辑后重放等操作，状态恢复逻辑更统一
   - 降低多轮对话后重生成导致时间线错乱的概率

2. **酒馆 Generate 的提示词来源边界更清晰**
   - 前端只保留自己负责收集和整理的上下文
   - 酒馆原生会提供的 `character / preset / worldbook` 不再重复塞入系统提示词
   - 让酒馆 Generate 和主 API 路径的行为边界更明确

## 版本 0.3.2 - 摘要系统与手机日记升级

### 新功能

1. **小总结 / 大总结双层记忆系统**
   - 每 5 轮完整对话自动生成 1 条小总结，按“用户发言 + 温婉回复”记为 1 轮
   - 每 50 条小总结自动生成 1 条大总结，用于长期上下文压缩
   - 小总结不会因为生成大总结而被删除，会继续保留在存档中

2. **摘要内容改为温婉日记风格**
   - 小总结改为以温婉第一人称写日记的方式生成
   - 摘要时会附带当前人设、地点、情绪、好感度、堕落度、性欲、穿着等状态信息
   - 大总结保留为长期记忆用的第三人称归纳，不使用日记口吻

3. **手机新增“温婉的日记”页面**
   - 手机日历页的“今日总结”卡片现在支持点击进入详情页
   - 新页面会展示全部小总结，并显示对应的游戏内日期和时间
   - 首页预览仍只显示最新一条总结，避免信息过载

4. **摘要生成状态通知**
   - 生成小总结或大总结时，前端顶部会显示悬浮提示
   - 提示会区分“小总结 / 大总结”的生成中、成功、失败状态
   - 通知样式调整为更接近手机通知的胶囊卡片，并增加图标与轻微入场动画

### Bug修复

1. **修复摘要失败后的处理逻辑**
   - 移除本地兜底摘要，不再在失败时写入低质量拼接结果
   - 小总结生成失败后不会推进 checkpoint，会在后续对话时继续补生成未总结内容
   - 大总结生成失败后同样不会推进 checkpoint，并会在下次新增小总结后重试

2. **修复对话历史摘要引用方式**
   - 对话生成时不再只简单截取最近消息，而是改为携带“未总结对话 + 最近 50 条小总结 + 全部大总结”
   - 减少旧消息丢失带来的上下文断层问题

3. **修复存档兼容与摘要时间缺失问题**
   - 存档系统兼容旧版字符串摘要和新版结构化摘要
   - 保存小总结时会同时保存对应的游戏内日期时间
   - 读档后会同步恢复小总结、大总结及各自的摘要进度

### 系统优化

1. **摘要触发规则重构**
   - 摘要统计从“角色消息条数”改为“完整对话轮次”
   - 让摘要批次更贴合真实聊天回合，避免半轮对话被错误计入

2. **手机摘要浏览体验优化**
   - 日历页默认仅展示当天最新一条总结
   - 想查看完整记录时可再进入日记页，信息层级更清晰

3. **手机与桌面端摘要入口统一**
   - 桌面布局与手机布局都会传递完整的小总结列表到手机界面
   - 保证不同显示模式下的日记内容一致

# 温婉0.2 更新日志

## 版本 0.2.2 - 精确位置系统

### ✨ 新功能

1. **精确位置系统（Precise Location System）**
   - **核心概念**：区分"大地点"和"精确位置"，实现更真实的寻找机制
   - **室内地点（家）**：范围小，100%能找到温婉
   - **大地点（学校、展会中心、港口等）**：范围大，即使在同一大地点也不一定能找到
   - **精确位置字段（exactLocation）**：温婉在大地点时的具体位置（如"cos社活动室"、"A展厅"、"游艇上"）
   - **可访问性字段（isAccessible）**：是否可被找到（如游艇已出海则false）

2. **概率性找到机制**
   - 学校：30%概率找到
   - 展会中心：20%概率找到
   - 港口：10%概率找到（可能已出海）
   - 商城：40%概率找到
   - 电影院：50%概率找到
   - 游乐场：35%概率找到

3. **通过微聊获取精确位置**
   - 玩家可以通过微聊询问"你在哪"
   - 温婉会回复精确位置（如"我在展会中心的A展厅"）
   - 知道精确位置后，前往该大地点可以100%找到温婉

### 🎮 玩法改进

1. **更真实的NTR场景**
   - 温婉在展会中心被NTR，玩家前往展会中心但找不到她
   - 等找到的时候，NTR事件可能已经结束
   - 增加了游戏的紧张感和真实感

2. **游艇出海场景**
   - 温婉坐游艇出海，玩家去港口也找不到（游艇已出海）
   - 只能通过微聊联系，无法直接找到

3. **学校场景**
   - 温婉在学校cos社活动室，玩家去学校不一定能找到
   - 需要知道具体位置或通过概率找到

### 🐛 Bug修复

1. **删除占位推特代码**
   - 问题：NTR事件发生时，如果AI未生成推特，会显示占位提示
   - 修复：删除占位推特逻辑，如果AI未生成推特则不显示任何内容
   - 现在完全依赖AI生成推特内容，不再显示系统占位提示

### 🔧 系统优化

1. **位置判断逻辑优化**
   - 更新远程互动判断，加入精确位置判断
   - 同一大地点但无精确位置信息 → 可能是远程（概率性找到）
   - 同一大地点且有精确位置信息 → 不是远程（可以找到）

2. **系统提示词更新**
   - 添加精确位置系统的详细说明
   - 告诉AI如何设置exactLocation和isAccessible字段
   - 告诉AI如何描述玩家在大地点中寻找温婉的过程

3. **数据结构扩展**
   - BodyStatus接口添加exactLocation和isAccessible字段
   - 响应格式规则更新，包含精确位置字段示例

## 版本 0.2.1 - 系统重构与优化

### 🐛 Bug修复

1. **修复跳过时间的好感度减少问题**
   - 问题：点击"跳过1天"时也会减少3点好感度
   - 修复：现在只有点击"跳过3天"才会减少好感度，"跳过1天"不再减少

2. **修复黄毛邀请判断逻辑**
   - 问题：高好感度（95）时，温婉仍然接受黄毛的邀请
   - 修复：加入好感度影响机制，好感度越高，温婉越倾向于拒绝黄毛邀请
     - 好感90+：拒绝（除非堕落>好感）
     - 好感70-89：80%概率拒绝
     - 好感50-69：50%概率拒绝
     - 好感30-49：20%概率拒绝
     - 好感0-29：接受

### ✨ 新功能

1. **自动时间流逝系统**
   - AI现在可以识别"睡觉"、"晚安"等指令
   - 自动描述时间流逝到第二天早上7点
   - 无需手动点击时间跳过按钮

### 🔧 系统优化

1. **简化行为系统**
   - 移除复杂的弧光系统（A/B/C/D/E）
   - 改用简洁的好感度+堕落度系统
   - 行为模式更直观、易理解

2. **代码清理**
   - 删除5个废弃的规则文件
   - 删除6个废弃的函数
   - 删除2个废弃的数据字段
   - 修复3处过时的弧光引用
   - 减少约600-800行废弃代码

3. **规则整合**
   - 整合重复的时间规则定义
   - 整合重复的好感度规则定义
   - 统一规则来源，避免重复

4. **世界书优化**
   - 删除所有弧光相关内容
   - 保留世界背景设定
   - 根据前端可访问的地点优化地点描述

### 📝 其他改进

- 丰富温婉的人设描述（外貌、性格、背景等）
- 优化系统提示词结构
- 提升代码可维护性
